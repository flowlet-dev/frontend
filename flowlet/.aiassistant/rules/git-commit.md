# Git Commit ルール

## ドキュメント概要

このドキュメントは、`Git Commit` に関するルールが記載されています。

## 目的

- コミット履歴を **あとから読んで理解できる** ようにする
- **変更理由・影響範囲・確認方法**を最小コストで残す

## 言語・基本方針

- コミットメッセージは **日本語**
- **1コミット=1目的**

## コミットメッセージ形式

```
<要約>

<変更内容の要点（任意・箇条書き可）>

<影響/確認方法/補足（任意）>
```

- 要約は **50～72文字** 目安
